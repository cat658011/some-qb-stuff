**qBittorrent Peer Guard**

Лёгкая автоматическая утилита на Python, которая очищает ваш qBittorrent от вредных, паразитных и бесполезных пиров — например, клиентов Xunlei/Thunder, нулевых личеров и мёртвых PEX-соединений.

### Возможности

- **Умная фильтрация пиров** — автоматически находит и блокирует проблемных пиров по разумным правилам.
- **Удобный табличный лог** — сохраняет информацию в красивый текстовый файл, который легко читать в любом редакторе.
- **Режим симуляции** (`--dry-run`) — можно сначала посмотреть, кого скрипт собирается банить, без реальных действий.
- **Тихий режим** (`--silent`) — идеально подходит для запуска в фоне через cron, systemd или screen.

### Установка

**Требования:**
- Python 3.8 или выше
- Библиотека `qbittorrent-api`

```bash
git clone https://github.com/cat658011/some-qb-stuff.git
cd some-qb-stuff
pip install qbittorrent-api
```

Не забудьте включить **Web UI** в qBittorrent:  
**Инструменты → Параметры → Web UI**

### Использование

```bash
python3 block-sus-peers.py [параметры]
```

Настройку можно делать как через флаги командной строки, так и через переменные окружения.

#### Основные параметры

| Параметр        | Описание                                              | По умолчанию          |
|-----------------|-------------------------------------------------------|-----------------------|
| `--host`        | Адрес WebUI qBittorrent                               | `127.0.0.1:8080`      |
| `--user`        | Логин WebUI                                           | `admin`               |
| `--pass`        | Пароль WebUI                                          | —                     |
| `--interval`    | Интервал между проверками (секунды)                   | 2.0                   |
| `--threshold`   | Максимум отданных байт нулевым пирам                  | 50 МБ                 |
| `--log-file`    | Путь к файлу лога                                     | None (отключено)      |
| `--silent`      | Отключить вывод в консоль                             | False                 |
| `--dry-run`     | Режим симуляции (без реальной блокировки)             | False                 |

### Примеры

**1. Обычный запуск с ведением лога**
```bash
python3 block-sus-peers.py --host 127.0.0.1:8087 --user admin --pass secret123 --interval 2.5 --log-file "./peers_base.txt"
```

**2. Проверка перед использованием (очень рекомендуется сначала)**
```bash
python3 block-sus-peers.py --dry-run
```

**3. Запуск в фоне (серверный вариант)**
```bash
export QBT_USER="your_username"
export QBT_PASS="your_password"

python3 block-sus-peers.py --host 10.0.0.5:8080 --interval 3 --log-file "/var/log/peer_guard.log" --silent
```

### Пример лога

```text
DATE TIME       | IP                    | FLAGS | PROG  | UPLOADED | DOWNLOADED | REASON                          | CLIENT
----------------|-----------------------|-------|-------|----------|------------|---------------------------------|--------------------
2026-05-16 00:18 | 117.84.128.123       | H X   | 0.0%  | 0 B      | 0 B        | BLACKLISTED_CLIENT              | Xunlei 11.2.1
2026-05-16 00:18 | 2803:f340:...        | H     | 0.0%  | 154 MB   | 0 B        | EXCESSIVE_UPLOAD_ZERO_PROGRESS  | FakeClient v1
```

### Правила фильтрации

Скрипт блокирует пиров по следующим критериям:

1. **EMPTY_CLIENT_NAME** — Пустое или скрытое имя клиента.
2. **BLACKLISTED_CLIENT** — Известные «плохие» клиенты (Xunlei, Thunder, xfwl, top-bt и подобные).
3. **EXCESSIVE_UPLOAD_ZERO_PROGRESS** — Личеры, которые много качают у вас, но остаются на 0% или вообще ничего не отдают.
4. **INACTIVE_PEX_CONN** — Мёртвые и зависшие PEX-соединения, которые просто занимают слоты.

**Важное замечание**: Правила №1 и №4 — самые сильные индикаторы ботов от Anti-P2P организаций.

### Зачем это нужно

На популярных раздачах часто сидит огромное количество ботов, фейковых клиентов и скрейперов. Они жрут лимиты соединений и upload-слоты, практически ничего не отдавая взамен.

В реальных тестах скрипт вычистил более **14 000** мусорных пиров всего за 30 минут. После этого очередь соединений очищается, скорость становится стабильнее, а нормальные пиры подключаются заметно лучше.
